<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin UI role mutation → profiles.update | client issues role changes; RLS profiles_admin_all is the server gate |
| direct URL → /admin/users | a non-admin can type the URL; RoleGuard + RLS must both deny |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-18 | Denial of Service | last-admin lockout via UsersPage | mitigate | UI guard: own-row select disabled when sole admin; cross-row demotion of the last admin blocked with a toast before any `.update` call (Task 1). DB trigger (Plan 02) backstops the Table-Editor / direct-API path |
| T-04-19 | Elevation of Privilege | billing/pending user reaching /admin/users by URL | mitigate | RoleGuard `allow={['admin']}` renders inline 403 (Task 3); `profiles_admin_all` RLS denies the underlying profiles read/update for non-admins server-side |
| T-04-20 | Tampering | stale role names allowing an invalid role write | mitigate | ROLES reconciled to admin/billing/pending (Task 1); the DB CHECK constraint rejects any other value even if the client is tampered |
| T-04-21 | Information Disclosure | ActivityPage querying a non-existent table leaking errors | mitigate | ActivityPage fully removed (Task 2) — no dead surface, no error-path information disclosure, no orphan route |
</threat_model>
