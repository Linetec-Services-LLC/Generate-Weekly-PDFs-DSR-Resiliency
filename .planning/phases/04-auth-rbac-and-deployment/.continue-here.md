### C. Portal code change (CLAUDE — route via `/gsd-quick`) — the ONLY repo change left
`portal-v2/src/components/auth/ResetPasswordPage.tsx` currently only waits for the
`PASSWORD_RECOVERY` event (implicit flow). With a `?token_hash=...` link it never fires,
so the page hangs on "Verifying your reset link…". Add token_hash handling in the mount
`useEffect` (keep the existing PASSWORD_RECOVERY listener as a fallback):

```ts
useEffect(() => {
  const params = new URLSearchParams(window.location.search);
  const tokenHash = params.get('token_hash');
  const type = params.get('type');
  if (tokenHash && type === 'recovery') {
    supabase.auth.verifyOtp({ token_hash: tokenHash, type: 'recovery' })
      .then(({ error }) => {
        if (error) setError('This reset link has expired. Request a new one.');
        else setReadyToReset(true);
      });
  }
  const { data: listener } = supabase.auth.onAuthStateChange((event) => {
    if (event === 'PASSWORD_RECOVERY') setReadyToReset(true);
  });
  return () => listener.subscription.unsubscribe();
}, []);
```
