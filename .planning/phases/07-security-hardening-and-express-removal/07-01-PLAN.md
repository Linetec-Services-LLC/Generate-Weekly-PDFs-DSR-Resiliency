## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| browser → Vercel Edge | Every HTTP response must carry SEC-02 security headers + CSP, applied at the CDN layer before the SPA is served. | All portal responses (HTML, JS, CSS) |
| browser → external origins (Supabase / hCaptcha / Sentry) | CSP allowlists exactly the origins the app uses; everything else is denied. | REST/Realtime/captcha/telemetry traffic |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-02-clickjack | Elevation of Privilege | All Vercel responses | mitigate | Ship BOTH `X-Frame-Options: DENY` and CSP `frame-ancestors 'none'` in the headers block (belt-and-suspenders for older browsers per RESEARCH Pitfall 3). Verified by Task 1 grep + Task 2 walkthrough. |
| T-07-02-xss | Tampering | SPA script execution | mitigate | CSP `script-src 'self' + hCaptcha origins` with NO `'unsafe-inline'`/`'unsafe-eval'` — possible because the Vite build emits only content-hashed external assets (RESEARCH §Question 1). |
| T-07-02-mime | Tampering | Downloaded Excel files / assets | mitigate | `X-Content-Type-Options: nosniff` prevents MIME-type confusion on served assets and signed-URL downloads. |
| T-07-02-downgrade | Tampering / Spoofing | Transport | mitigate | `Strict-Transport-Security: max-age=63072000; includeSubDomains` forces HTTPS; `preload` deliberately omitted (operator-only, permanent). |
| T-07-02-referrer-leak | Information Disclosure | Outbound navigations | mitigate | `Referrer-Policy: strict-origin-when-cross-origin` strips path/query from cross-origin referrers. |
| T-07-02-csp-breakage | DoS (self-inflicted) | Live app via enforcing CSP | mitigate | Ship `Content-Security-Policy-Report-Only` FIRST (this plan); the enforcing flip (07-03) is gated on the Task 2 zero-violation walkthrough. An over-tight CSP can never silently break a live origin. |
| T-07-02-sentry-region | DoS (telemetry loss) | Sentry ingest | accept→verify | Sentry region is unknown from the repo (RESEARCH A1). Default US pattern `*.ingest.sentry.io`; Task 2 step 7 confirms events still flow; operator switches to `*.ingest.de.sentry.io` if EU. |
</threat_model>
