<objective>
Provision the Supabase data plane that the whole v1.1 portal depends on:
the `public.artifacts` metadata table, the `public.profiles` role table,
role-aware Row-Level Security on both tables PLUS `storage.objects`, and a
private `excel-artifacts` Storage bucket. After this plan, access control is
real from day one — anonymous and `pending`-role callers get zero artifact
rows, while `admin`/`billing` roles can read rows and (in later phases)
generate signed download URLs.

downstream SEC-* / RBAC-* requirements. The DDL is committed in-repo (D-03)

`supabase db push` / CLI-migration task (this project applies SQL by hand,
mirroring `billing_audit/schema.sql`).

         CREATE POLICY profiles_self_read ON public.profiles
             FOR SELECT TO authenticated USING (auth.uid() = id);
         CREATE POLICY profiles_admin_all ON public.profiles
             FOR ALL TO authenticated
             USING (public.current_user_role() = 'admin')

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| anonymous/public → PostgREST `/rest/v1/artifacts` | Untrusted public reads cross here; the anon key is intentionally public, so the DB (RLS) must be the guard. |
| authenticated `pending` user → artifacts/storage | A logged-in but unapproved user must get zero billing data. |
| authenticated `admin`/`billing` → storage.objects | Authorized read; createSignedUrl validates against the Storage SELECT policy. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-rls-anon | Information Disclosure | `public.artifacts` SELECT | mitigate | No policy for `anon`; RLS enabled; anon-curl `[]` proof (Task 2 step 6). |
| T-03-rls-pending | Elevation of Privilege | `public.artifacts` SELECT | mitigate | Policy JOINs `public.profiles` and requires `role IN ('admin','billing')`; `pending` gets 0 rows. NEVER `USING (true)` (verify gate forbids it). |
| T-03-storage-signurl | Information Disclosure | `storage.objects` SELECT | mitigate | Role-gated `storage_artifacts_role_select` policy on `bucket_id='excel-artifacts'`; required for createSignedUrl; without it even valid sessions 403 (Pitfall B). |
| T-03-public-bucket | Information Disclosure | Storage bucket | mitigate | Bucket created `public:false` (Task 2 step 4); downloads only via short-lived signed URLs (Plan 03 / Phase 05); never `getPublicUrl()`. |
| T-03-variant-check | Denial of Service (data loss) | `public.artifacts.variant` | accept | `variant TEXT` no CHECK by design (forward-compat); a bad token is caught at the app layer in Plan 02, not silently dropped by a DB CHECK. |
</threat_model>
