## Phase 06 Constraints (Carried from Phase 05 Review)

These constraints are LOCKED — the planner and executor must address them during
polish rather than deferring to Phase 07:

### C-01: Single Global Toast Region (WR-05)

**Problem:** `App.tsx` renders a `<ToastContainer>` (line 81) backed by its own
`useToast()`. `ArtifactTable.tsx` also renders a second `<ToastContainer>` (line
259) backed by its own `useToast()`. Two independent stacks exist.

**Required fix in Phase 06:** Lift `useToast` into a React context (or adopt a
singleton toast library such as `sonner`). `ArtifactTable` must consume the
shared context and NOT render its own `<ToastContainer>`. The Realtime
"N new artifacts" toast and the download-error toast must both appear in the
SAME stack — bottom-right, single z-50 layer. Remove the local
`<ToastContainer>` from `ArtifactTable.tsx`.

**Design implication:** The Realtime `info`-variant toast and the download
`error`-variant toast coexist in one queue. Toast ordering: newest on top
(current behavior, preserve it).

### C-02: Variant Options Query Cap (WR-03)

**Required fix in Phase 06:** Add `.limit(2000)` to the `['artifact-variants']`
query and set `staleTime: 10 * 60 * 1000` (10 min). This is a data-layer fix
with no visible UI change, but it is part of the polish pass.

## Spacing Scale

Tailwind default 4px-based scale (confirmed from `tailwind.config.ts` — no
custom spacing overrides declared). Every declared spacing value in this spec is
a multiple of 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`p-1`, `gap-1`) | Icon gaps, inline tight padding |
| sm | 8px (`p-2`, `gap-2`) | Compact element spacing, badge padding |
| md | 16px (`p-4`, `gap-4`) | Default element spacing, card padding |
| lg | 24px (`p-6`, `gap-6`) | Section padding, table cell padding |
| xl | 32px (`p-8`, `gap-8`) | Layout gaps, page-level padding (lg+) |
| 2xl | 48px (`py-12`) | Empty-state vertical breathing room |
| 3xl | 64px | Major section breaks (not used in table UI) |

**Exceptions (multiples of 4 outside the standard token set — deliberate, documented):**
- `px-5` / `py-3` (20px / 12px) on table header AND data cells — carried from the
  Phase 05 `UsersPage.tsx` / `ArtifactTable.tsx` canonical pattern; both are
  multiples of 4. Kept for visual continuity with the already-shipped table; row
  height stays at the 56px virtualizer estimate.
- `min-h-[44px]` on the mobile Download button — NOT a spacing/padding value but a
  sizing constraint that fulfils the WCAG 2.5.5 (Target Size) 44px minimum
  touch-target guarantee (44 = 4 × 11). The button's own padding is `py-3 px-4`
  (12px / 16px, both multiples of 4); `min-h-[44px]` only floors the rendered
  height so the target is never smaller than 44px regardless of line-height.

## Typography

| Role | Size | Weight | Line Height | Tailwind |
|------|------|--------|-------------|---------|
| Body | 14px | 400 (regular) | 1.5 (leading-normal) | `text-sm` |
| Label / meta | 12px | 400 (regular) | 1.4 (leading-snug) | `text-xs` |
| Column header | 12px | 600 (semibold) | 1.4 (leading-snug) | `text-xs font-semibold` |
| Heading | 24px | 600 (semibold) | 1.2 (leading-tight) | `text-2xl font-semibold` |

**Notes:**
- Exactly 3 distinct sizes in use: 12px (label/header), 14px (body), 24px (heading).
- 2 weight values: 400 (regular — body, meta) and 600 (semibold — all emphasis
  incl. the page heading). At 24px the heading is already dominant over 14px
  body / 12px label by size alone, so no heavier weight is needed.
- Emphasis (600 / `font-semibold`) is used for: the page heading, column headers,
  the WR # in mobile cards, and the count inside the "Load new" pill / toast.
- Toast message text: `text-sm text-slate-700` (14px, weight 400) — preserved from
  existing `Toast.tsx` implementation.
- "N new artifacts" in toast/pill uses body size (14px); the numeric count is
  shown in `font-semibold` (600).

## Component Inventory — New in Phase 06

The following net-new components must be built. Use the existing primitive and
pattern library as structural templates.

### 1. `useRealtimeArtifacts` hook

**Location:** `src/hooks/useRealtimeArtifacts.ts`

**Contract:**
- Subscribes to `supabase.channel('artifacts')` with `.on('postgres_changes', { event: 'INSERT', schema: 'public', table: 'artifacts' }, callback)`.
- On INSERT: increments a `pendingCount` (number) counter; does NOT insert rows
  into the existing TanStack Query cache mid-scroll.
- Returns `{ pendingCount, clearPending }`.
- `clearPending` resets `pendingCount` to 0 AND calls
  `queryClient.invalidateQueries({ queryKey: ['artifacts'] })` to refetch.
- On unmount: calls `channel.unsubscribe()` — zero subscription leak.

### 2. `NewArtifactPill` component

**Location:** `src/components/artifacts/NewArtifactPill.tsx`

**Contract:**
- Props: `count: number`, `onLoad: () => void`, `onDismiss: () => void`
- Visible only when `count > 0`.
- Desktop (sm+): positioned `sticky top-0 z-10` above the table card, centered
  or right-aligned. Uses L2 glass: `backdrop-blur-sm bg-white/80 border
  border-slate-200 shadow-md rounded-full`.
- Mobile (<sm): full-width `bg-brand-red text-white` banner, `rounded-none` or
  `rounded-lg`, pinned above the artifact card list.
- Contains two affordances:
  - "Load N new artifact(s)" button — calls `onLoad`, then pill dismisses.
  - "×" dismiss button — calls `onDismiss` without loading.
- Framer Motion: `initial={{ opacity: 0, y: -8 }} animate={{ opacity: 1, y: 0 }}
  exit={{ opacity: 0, y: -8 }} transition={{ duration: 0.2 }}` via `AnimatePresence`.

**Copywriting:**
- 1 new: "Load 1 new artifact"
- 2+: "Load {N} new artifacts"
- Dismiss aria-label: "Dismiss new artifact notification"

### 3. Mobile card view — `ArtifactCard` component

**Location:** `src/components/artifacts/ArtifactCard.tsx`

**Contract:**
- Rendered ONLY at `<640px` viewport width (Tailwind `sm:hidden`). The
  existing virtualized `<table>` layout is `hidden sm:block`.
- Each artifact becomes a `GlassCard` (L1 styling: `bg-white rounded-2xl
  border border-slate-100 shadow-sm`; NOT the full `backdrop-blur` glass —
  save L2 for floating elements).
- Card layout (vertical stack, `p-4 space-y-2`):
  - Row 1: WR # in `text-sm font-semibold text-slate-900` + week-ending date
    in `text-xs text-slate-500` — always visible.
  - Row 2: Variant `<Badge>` (existing variants: success=primary, info=AEP,
    default=base) + file size in `text-xs text-slate-500`.
  - Row 3: Created date in `text-xs text-slate-500` + Download button.
- Download button: `min-h-[44px] py-3 px-4` (44px WCAG 2.5.5 floor; padding
  12px / 16px — all multiples of 4), `bg-brand-red text-white rounded-lg
  text-sm font-medium`. Loading state: spinner icon replaces download icon
  (use `lucide-react` `Loader2` with `animate-spin`).
- `React.memo` wrapped at module level (same Pitfall 3 rule from Phase 05).

### 4. `ToastContext` (replaces dual-stack — C-01 fix)

**Location:** `src/contexts/ToastContext.tsx`

**Contract:**
- `React.createContext` + provider wrapping the app tree (inside
  `QueryClientProvider`, outside `BrowserRouter` or immediately inside it —
  matches the existing `AuthProvider` nesting pattern).
- Exports `useToastContext()` hook returning `{ toasts, addToast, removeToast }`.
- Single `<ToastContainer>` rendered inside the provider (or in `App.tsx`,
  consuming the context).
- Remove `useToast()` calls and `<ToastContainer>` from `ArtifactTable.tsx`.
- `useDownloadArtifact` receives `addToast` as a prop (already threaded in;
  just source it from context rather than a local `useToast()`).

## Animation Contract

All animations use `framer-motion` 11.11. All durations and easing are
prescriptive — do not deviate without explicit approval.

### Reduced Motion

`globals.css` already sets `animation-duration: 0.01ms !important` for
`prefers-reduced-motion: reduce`. This is the global override. All Framer Motion
variants MUST also check `useReducedMotion()` from `framer-motion` and skip or
minimize animations when it returns `true`:

```typescript
const prefersReduced = useReducedMotion();
const transition = prefersReduced ? { duration: 0 } : { duration: 0.2 };
```

### Animation Catalog

| Element | Variant | Duration | Easing | Notes |
|---------|---------|----------|--------|-------|
| Page transitions (existing `PageTransition`) | opacity 0→1, y 8→0 | 300ms | ease-out | Preserve existing — do not change |
| Row entrance stagger (initial load only) | opacity 0→1 | 150ms | ease-out | Stagger delay: index × 20ms, capped at 200ms (10 rows max stagger) |
| Row entrance (subsequent scroll loads) | NO animation | — | — | Do not animate rows fetched by infinite scroll — only initial page load |
| Toast in/out (existing) | x 100→0 / x 0→100, opacity | Spring (stiffness 300, damping 25) | — | Preserve existing `Toast.tsx` behavior |
| "Load new" pill appear/disappear | opacity 0→1, y -8→0 | 200ms | ease-out | `AnimatePresence` controlled |
| Mobile card entrance | opacity 0→1, y 4→0 | 150ms | ease-out | On initial load only; no stagger on scroll |
| Download button loading state | spinner rotation | 600ms linear infinite | — | `lucide-react` `Loader2 className="animate-spin"` |
| View transitions (existing `PageTransition`) | opacity | 300ms | — | No change |

### Performance Constraints

- `motion.tr` stagger MUST use `AnimatePresence` with `initial={false}` on the
  parent wrapper to prevent re-animating rows that are already visible when
  filters change. Animation plays ONCE on initial mount of the row batch.
- The virtualizer renders rows as `<div>` wrappers with absolute positioning
  (Phase 05 pattern). Phase 06 may NOT change the virtualizer container structure.
  Row entrance animations go on the inner `ArtifactTableRow` component's root
  element, not on the virtualizer's positioning wrapper `<div>`.
- No layout animations (`layout` prop) inside the virtualized scroll container —
  layout thrash would fire on every scroll tick. The `layout` prop is ONLY
  acceptable on the `ToastContainer` stack (already uses `layout` on individual
  toasts — preserve).
- Desktop row entrance: `will-change: opacity` is acceptable; `will-change: transform`
  is ONLY acceptable on the Download button spinner (bounded count).

## Accessibility Contract

All of the following must be TRUE at phase close (maps to UI-03 / WCAG AA):

### Keyboard Navigation

| Element | Required behavior |
|---------|------------------|
| Search bar | Tab-reachable, visible focus ring (`focus-visible:ring-2 focus-visible:ring-brand-red/50`) |
| Variant filter chips / checkboxes | Tab-reachable; selected state communicated via `aria-pressed` or `aria-checked`; `Enter`/`Space` toggles |
| Column sort headers | Tab-reachable (`tabIndex={0}`); `Enter`/`Space` triggers sort; `aria-sort="ascending"/"descending"/"none"` |
| Download button | Tab-reachable; `aria-disabled="true"` when downloading |
| "Load new" pill | Tab-reachable; `Enter`/`Space` loads; `Escape` dismisses (or explicit × button) |
| Toast dismiss | Tab-reachable; `aria-label="Dismiss"` (existing — preserve) |

### ARIA Roles

| Element | Role |
|---------|------|
| Table (desktop) | `role="table"` on wrapper; `role="columnheader"` on header cells; `role="row"`/`role="cell"` on data rows (existing Phase 05 — verify and preserve) |
| Mobile card list | `role="list"` on container; `role="listitem"` on each `ArtifactCard` |
| "Load new" pill | `role="status"` with `aria-live="polite"` so screen readers announce new artifact arrival |
| Toast region | `role="status"` or `role="alert"` (error toasts); `aria-live="assertive"` for errors, `"polite"` for info/success |

### Color Contrast (WCAG AA — minimum 4.5:1 for normal text, 3:1 for large)

| Pair | Approximate ratio | Status |
|------|-------------------|--------|
| `text-slate-900` on `bg-white` | ~21:1 | Pass |
| `text-slate-700` on `bg-white` (toast body) | ~9:1 | Pass |
| `text-slate-500` on `bg-white` (meta text) | ~4.6:1 | Pass (AA) |
| `text-white` on `bg-brand-red` (#C41230) | ~5.1:1 | Pass (AA) |
| `text-brand-red` on `bg-white` (#C41230 on white) | ~5.1:1 | Pass (AA) |
| `text-slate-400` on `bg-white` | ~2.8:1 | FAIL for normal text |

**Action for `text-slate-400`:** Upgrade secondary meta text to `text-slate-500`
in all new Phase 06 components (`ArtifactCard`, `NewArtifactPill`) — already
reflected in the `ArtifactCard` contract above. Existing Phase 05 uses of
`text-slate-400` should be evaluated; upgrade where text carries meaning (dates,
sizes) and leave decorative uses (icon color, separator) as-is.
