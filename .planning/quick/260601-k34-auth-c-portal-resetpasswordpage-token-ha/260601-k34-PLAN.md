# Quick Task 260601-k34 ‚Äî ResetPasswordPage token_hash recovery flow (auth-C)

## Objective

`portal-v2/src/components/auth/ResetPasswordPage.tsx` today only reacts to the
implicit-flow `PASSWORD_RECOVERY` `onAuthStateChange` event. With the corrected
Supabase email template (`{{ .RedirectTo }}?token_hash=...&type=recovery`) the
recovery link uses the **PKCE/token_hash flow**, so `PASSWORD_RECOVERY` never
fires and the page hangs forever on "Verifying your reset link‚Ä¶".

Add a `verifyOtp({ token_hash, type: 'recovery' })` path on mount while keeping
the `PASSWORD_RECOVERY` listener as a fallback for the implicit flow.

This is the **only remaining repo code change** for Phase 04 plan 04-06.
Everything else (Supabase template, redirect allow-list, Vercel promote) is
user-side dashboard work.
